# 피트니스 훈련 프로그램 어플리케이션

피트니스 훈련 프로그램을 위한 서비스 개발입니다.

1인 개발이기 때문에 최소한의 기능만 넣겠습니다.

MVP 수준의 기능만 필요합니다.

시장 검증이 우선입니다.

# 기술 스택

Framework: Next.js 16 (App route)

UI Kit: Shadcn/ui + Tailwind CSS (직접 디자인하는 시간을 80% 줄여줍니다)

Backend/DB: Supabase (Auth, Database, Storage를 따로 구축할 필요 없이 즉시 사용 가능)

ORM: drizzle orm

Payment: TOSS

Subscription: ??

Deployment: Vercel (배포 및 CI/CD 자동화)

# 빠른 개발을 위한 전략

1. 하나의 Next.js 프로젝트 안에서 폴더 구조로 역할을 나누는 전략
   app/
   ├── (marketing)/ # 마케팅 사이트 (Landing Page)
   │ ├── page.tsx # fitness.com/
   │ └── layout.tsx # 브랜드 비주얼 중심 레이아웃
   ├── (coach)/ # 코칭 SaaS (Admin)
   │ ├── dashboard/ # fitness.com/dashboard
   │ ├── programs/ # 프로그램 빌더
   │ └── layout.tsx # 사이드바 중심 관리자 레이아웃
   └── (user)/ # 유저 웹
   ├── shop/ # fitness.com/shop (결제)
   └── layout.tsx # 모바일 하단 탭바 중심 앱 레이아웃

2. 역할 기반 접근 제어 (RBAC)
   하나의 프로젝트이므로 보안이 중요합니다. middleware.ts 파일을 하나만 작성하면 모든 경로를 통제할 수 있습니다.

# 일정

3. 수정된 6주 초압축 일정 (Role 중심)주차개발 타겟핵심 작업 내용

1-2주데이터 & 코치DB 설계(Supabase), 코칭 프로그램 빌더(내용 업로드), 공통 인증
3주유저 웹마케팅 페이지 완성, 정기 결제 연동, 구독 관리 페이지4주유저 앱(PWA)모바일 전용 UI 개발, 오늘의 훈련 리스트, 완료 체크 로직
5주기록 & 로그훈련 일지 기록(사진/텍스트), 과거 히스토리 조회, 푸시 알림(PWA)
6주통합 & 배포전체 시나리오(결제→훈련→기록) 테스트, Vercel 최종 런칭

# 라우팅

기본적인 앱구조 입니다.

app/
├── (marketing)/ # [1] 브랜드 마케팅 사이트
│ ├── layout.tsx # 투명 헤더, 와이드 레이아웃
│ ├── page.tsx # 히어로 섹션, 서비스 소개
│ ├── pricing/ # 가격 정책 페이지
│ └── reviews/ # 고객 후기(Social Proof)
│
├── (coach)/ # [2] 코치 SaaS (데스크탑 최적화)
│ ├── layout.tsx # 왼쪽 사이드바 네비게이션 적용
│ ├── dashboard/ # 전체 회원 현황 대시보드
│ ├── programs/ # 프로그램 관리
│ │ ├── page.tsx # 리스트 보기 (CRUD)
│ │ ├── new/ # 신규 생성
│ │ └── [id]/edit/ # 프로그램 빌더 (스케줄링, 영상 업로드)
│ ├── members/ # 회원 관리
│ │ ├── page.tsx # 전체 회원 목록
│ │ └── [userId]/ # 개별 회원 진행률 & 기록 확인
│ └── notices/ # 공지사항 발송/관리
│
├── (user-web)/ # [3] 유저 상점 & 구독 관리 (반응형)
│ ├── layout.tsx # 일반 웹 헤더/푸터 적용
│ ├── shop/ # 프로그램 탐색 및 리스트
│ ├── checkout/ # 결제 페이지 (PortOne 연동)
│ └── subscription/ # 내 구독 관리 (해지, 카드변경)
│
└── (user-app)/ # [4] 유저 전용 훈련 앱 (PWA 전용)
├── layout.tsx # 모바일 하단 탭바 레이아웃 (App-like UX)
├── today/ # 오늘의 훈련 (체크리스트)
├── training/[id]/ # 상세 훈련 내용 (영상, 가이드)
├── journal/ # 훈련 기록 (사진 업로드, 로그)
│ └── history/ # 내 과거 기록 히스토리
└── profile/ # 소셜 로그인 정보 및 설정

## 마케팅 사이트

- 가게로 치면 외관에서 보는 인테리어나, 매장의 디자인이다
- 처음으로 보이는 부분이기 때문에 디자인을 신경써야한다
- 고객이 아닌사람에게 는 완성도가 없어보일수있따.
- 완성도와 직결된다.
